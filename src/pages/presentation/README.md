### 3.2 工具生态：按「过去 / 现在 / 未来」理解

> 同一件事（写代码 / 做事情），不同阶段的工具关注点不同。

#### 过去：Cursor / Trae（AI IDE / 端到端工程师）

**Cursor - AI 原生 IDE**
- 基于 VS Code 二次开发
- 支持多模型：OpenAI、Anthropic、Gemini、xAI、Cursor
- 完整代码库理解能力
- 自主性滑块：从 Tab 补全到完全自主 agent

**三种使用模式：**
1. Tab 补全 - 像写代码一样自然
2. Cmd+K - 针对性编辑
3. Agent 模式 - 完全自主（构建、测试、演示）

**实际效果：**
- NVIDIA：40,000 工程师全面使用
- Salesforce：90% 开发者使用，PR 速度/质量双位数提升
- Y Combinator：最佳工程师都在用

**Trae - 你的 10x AI 工程师**
- Understand. Execute. Deliver.（理解 → 执行 → 交付）
- 强调端到端交付：理解需求 → 独立编码 → 完整交付

---

#### 现在：Claude Code（终端里的编程搭档）

**为什么是 Claude Code？**
- 代码能力强、上下文理解好
- 不是补全工具，而是工程搭档（读仓库 / 跑命令 / 产出 diff / 验证闭环）

**实战演示方向：**
- 环境部署
- 常见问题
- 实际案例

---

#### vibe coding 的最佳实践（把 AI 变成“可复利的系统”）

- 让 AI 帮你快速实现想法
- 不要追求完美，先跑起来
- 在迭代中学习 AI 的思维方式

> 原则：把一次性对话，变成可重复的流程。

### 3.3 OpenClaw：从 IDE 到个人助理（压轴）

> Claude Code 帮你写代码，OpenClaw 帮你做一切。

#### 3.3.1 什么是 OpenClaw？

**定位：**
- 个人 AI 助手框架
- 不止写代码，还能操作你的“数字生活”
- 24/7 待命，随时响应

**核心能力：**
- **消息与沟通：** 多渠道接入（Telegram/Discord/飞书/WhatsApp/Slack/Signal…），主动检查与提醒
- **文件与知识：** 本地文件读写、飞书文档、长期记忆管理
- **自动化能力：** Shell、浏览器自动化、节点控制（手机/摄像头/屏幕…）、定时任务（heartbeat / cron）

---

#### 3.3.2 模型切换：GLM-4.7 → ChatGPT 5.2（性格大变）

同一套使用方式，换模型之后“助理的性格”会明显变化：

- **GLM-4.7：** 更像一个美国人（更直接、更快、更偏执行）
- **ChatGPT 5.2：** 更像一个德国人（更严谨、更爱确认、更重视边界）

**结论：** 不是你变了，是模型的“气质/偏好”变了。

---

#### 3.3.3 真实案例：从「搞坏 WSL」到安卓手机上的 Dr.Two

我从今年 2 月初开始用 OpenClaw，到现在刚好一个月。

- **第一世：WSL Ubuntu**
  - 配置很快，但也很容易“用坏”
  - 让它做任务时把配置文件搞乱 → 最后整个 WSL 直接起不来

- **第二世：安卓手机 Termux（Dr.Two）**
  - 吃过教训后形成机制：
    1) 配置修改前先备份
    2) 重要修改先在测试环境试
    3) 建立守护进程监控 gateway 状态

---

#### 3.3.4 最近一次折腾：更新版本 + 换模型 → 行为大变

这段其实就是 3.3.2 的“现实版”：

- 因为 **OpenClaw 更新版本** + **更换模型（GLM-4.7 → ChatGPT 5.2）**，助理的行为习惯突然变了：
  - 以前更“直接执行”
  - 现在更“严谨确认”（每一步都想问一句）

我们真正做的不是“吐槽模型”，而是把它重新调回可用：

- 找到导致频繁确认/阻塞的配置开关
- 把不需要的确认开关关掉，把需要的边界留住
- 最终让它恢复到“该干活就干活”的状态（昨晚才调到正常）

**这一段的 vibe coding 含义：**
> 工具会变、模型会变，但你的系统要通过配置和文档把“稳定性”固化下来。

---

#### 3.3.5 上限极高，下限羸弱（以及为什么它仍然是未来）

- **上限极高：** 一旦开始用，会不断发现新可能性（发邮件 → 监控服务 → 管理工作流）
- **下限羸弱：** 配置文件错了就崩，一句话没说清就做错事

关键是：**这是一个会成长的东西**——用得越久、规则越清楚、文档越完善，它就越像你的专属助理。

2. **快速上手**（今天就用起来）

## 5. 或者让AI提建议

同一个 Prompt，不同的人，会得到完全不同的答案。

> **"2026 年，如果只选一个习惯——只要做到每天稳定执行，就会最大程度上改变你的人生——这个习惯是什么？"**

你会得到一个更贴近你背景的建议，然后你要做的是：
- 把它拆成「每天都能做完」的小动作
- 把结果固化成**可复用的东西**（代码 / 笔记 / 脚本 / 流程 / 文档）

这就是 vibe coding 的落地：**每天产出一个可积累的确定性成果**。
