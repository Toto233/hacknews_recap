## 过去：Cursor + Trae

**关键词：** IDE 里把 AI 变强，把写代码这件事加速

- 低门槛：上手快、立刻见效
- 高频：日常写代码随时用
- 适合：原型开发 / 日常迭代 / 中大型项目

## Cursor - AI 原生 IDE

**核心特点：**
- 基于 VS Code 二次开发
- 支持多模型：OpenAI、Anthropic、Gemini、xAI、Cursor
- 完整代码库理解能力

**三种模式：**
1. Tab 补全 - 像写代码一样自然
2. Cmd+K - 针对性编辑
3. Agent 模式 - 完全自主

## 现在：Claude Code + Codex CLI

**关键词：** 终端里的工程师搭档（读仓库 / 跑命令 / 生成 diff / 跑测试）

- 更贴近团队工程流程：命令 + diff + 验证闭环
- 适合：修 bug、重构、做可复现的改动

---

## Claude Code - 为什么选择它？

**独特优势：**
- 代码能力强，上下文理解好
- 不是简单的补全工具，而是真正的编程搭档
- 可以理解整个项目

---

## Claude Code - 最佳实践

- 让 Claude 帮你快速实现想法
- 不要追求完美，先跑起来
- 在迭代中学习 AI 的思维方式

---

## Codex CLI（现在）

**是什么：** OpenAI 的命令行编程助手（终端里读/改代码、跑命令、生成补丁）

**适用场景：**
- 快速理解仓库、定位问题
- 跑测试/构建 → 根据输出迭代修复
- 产出可审阅的 diff / PR 说明

**一句话：** 更强调「工程闭环」而不是“只写几段代码”。

## 💡 CLAUDE.MD 文件的重要性

**CLAUDE.MD 是 Claude Code 的"大脑"，写得越精确，效果越好。**

**关键原则：**

1. 明确技术栈和约定
2. 指定工具和命令
3. 代码风格约定
4. 项目结构说明

## 关键习惯：及时更新，固化知识

**把它们写下来，固化到 MD 文件里：**

- 发现了好用的工具 → 写进 `TOOLS.md`
- 学到了新技巧 → 写进 `AGENTS.md`
- 遇到问题 → 写进 `MEMORY.md`

**这就是 vibe coding 的本质：工具在帮你，你也在喂工具。**
