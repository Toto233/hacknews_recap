## 1. 特斯拉终结S/X传奇！弗里蒙特工厂变身百万Optimus机器人军团 (Tesla ending Models S and X production)

特斯拉CEO埃隆·马斯克在第四季度财报电话会上宣布，公司将“光荣退役”Model S和Model X车型的生产线，转而利用加州弗里蒙特工厂打造Optimus人形机器人年产能百万的生产线。这对科技爱好者来说，真是激动人心的大新闻！Model S自2012年上市、Model X 2015年问世，是特斯拉继首款Roadster后的最老车型，近年来面临全球电动车竞争加剧，价格已大幅下调，目前官网起售价分别为9.5万美元和10万美元。但它们销量有限，去年特斯拉全球交付159万辆车中，Model 3和Y占97%，后者起价仅3.7万和4万美元，且去年底推出更亲民版本。

尽管本季度盈利超预期，特斯拉却录得史上首次年度销量下滑，四季度前三季度已有三季度负增长。马斯克正将焦点从传统电动车转向无人驾驶和人形机器人——Optimus瞄准工厂劳作到保姆等全能场景，本季度将亮相第三代“量产版”设计。新供应链从零起步，工厂还将扩招员工、大幅提升产量。

## 2. 邮件竟有“500英里诅咒”！老bug让大学邮箱瞬间穿越回马车时代 (We can’t send mail farther than 500 miles (2002))

![邮件竟有“500英里诅咒”！老bug让大学邮箱瞬间穿越回马车时代 ](D:\python\hacknews\output\images\20260130\We_can’t_send_mail_farther_than_500_miles_(2002).png)

几年前，美国北卡罗来纳研究三角区一所大学的统计系主任打电话给校园邮件系统管理员，报告一个匪夷所思的问题：他们无法发送超过500英里（约800公里）外的电子邮件！更精确地说，是520英里以内偶尔能发，超出就彻底失败。

管理员起初以为是玩笑，但系主任一本正经：他们等了几天，收集数据后请地统计学家绘图确认——邮件发送半径刚好卡在520英里。纽约（420英里）能到，普罗维登斯（580英里）不行；本地服务器发往西雅图ISP失败，却能发给北卡本地人。统计学家们不愧专业，先攒数据再报警，管理员听得头大。

登录服务器测试，问题真实可复现：sendmail.cf配置正常，竟是管理员自己写的旧版；telnet SMTP端口，弹出SunOS Sendmail 5的横幅！原来顾问最近打补丁重启服务器，意外恢复了Sun系统自带的古老Sendmail 8前版本，这个bug版竟内置“距离限制”——可能是早期测试遗留或奇葩配置，导致远距离邮件“物理”失效。

这科技乌龙太有趣！凸显老软件隐患：一补丁，就能让现代邮件“回归马车时代”。
